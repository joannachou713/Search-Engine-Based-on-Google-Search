Motivation

動機

當想要找餐廳聚餐時，菜單通常會是形成最後決定的重要因素。但有時候 輸入餐廳名稱後，點進去第一條連結辛苦滑到底卻發現並沒有菜單，只能 在關鍵字後面加上「菜單」兩個字重新尋找。所以我們希望能做出一個專 門搜尋菜單的系統，主要從別人的食記去查詢，因為有些店不一定會有官 網的菜單，再依照時間先後排序食記，讓最新的菜單能出現在最前面。最 後，使用者只需要輸入餐廳名稱即可確定能在第一個結果中找到菜單，而 不需要再浪費一些時間自行在食記裡尋找。
Motivation : When finding restaurants for meals, menus are always one of the important factors to make the decision. But sometimes we enter the restaurant name and click in the first link, the menus aren’t exist. The only solution is searching again by adding a word ”menu” after the keyword. So we want to develop a menu system, which search from dining briefs (some restaurants don’t have official menu) on net and order them by time, and thus the earliest menu can appear in the front.

Tricks

技巧

在搜尋到的網頁中，設定條件要先是特定網站 ( 食記類型 ) ，接下來搜 尋關鍵字 ( 例如 : 菜單、menu、美食、餐廳、 價格 )，再來是看網站圖 片的數量，多的排在前面，最後 依照日期新舊排序。
Search Tricks: In all the websites, we first search for the websites that are related to restaurant introduction. Next, we search for websites that contains the most keywords(such as menu, food, price, restaurant). Third, we check the pictures in the websites and put them in order(from the most to the least). Last, put the latest articles in the top of the list.

System Introduction

系統介紹

使用者 : 想找餐廳並想先看菜單的人 User: People who want to search for a restaurant and see its menu first. 目的 : 輸入餐廳即可獲得相關資訊 ( 菜單 ) Purpose: Users can type in restaurants’ name and get their menu. 在使用者輸入關鍵字之後，程式會將此輸入關鍵字加上預設關鍵字從 Google 取得搜 尋結果連結; 取得 Google 的搜尋結果連結之後，會在 SearchResult存結果的陣列中為每個連結新增 URLobj的物件， 存取 URL 及網頁標題與網頁的關鍵字、圖片、 時間分數，並在每個 URLobj 內新增一個 Counter 物件， 幫助 URLobj 算出此網頁 的三項分數，而時間的算法是把各種部落格的時間表示法格式化，轉成 YYMMDD 的格 式，並將小數點放在年與月中間，避免過度影響總分。
When the users types keywords, we will put these with default keyword to get Google connections . After that , we put these URL into URLobj object from SearchResult array, which stores the websites title, the amount of keyword and image has appeared, and the time score. Then, add a Counter object in every URLobj to help URLobj count the websites’ three scores. The calculation of time is determined by formatting every blogs’ time state to a standard way(YYMMDD), and put decimal between year and month in order to avoid over-influences by month.

System Introduction

系統介紹

在 Counter 算完個別分 數之後回傳到 URLobj 算總分，其中公式是 (imgcount * 2 + keywordcount * 1.5 + timecount * 1) / 4.5，在算好成績之後，依照成績高低排序存在 SearchResult 裡存 有所有 URLobj 的 ArrayList，而此排序好的陣列就是我們的搜尋結果。
When Counter has calculated all individual scores, then return three scores to URLobj to count total score which different items have different weight (imgcount * 2 + keywordcount * 1.5 + timecount * 1) / 4.5. Sort the total score from high to low in SearchResult by ArrayList type, this ArrayList will be our finally search results.
